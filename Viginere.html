The Viginere Cipher

History

Are you ready for the hard one? I hope so! This cipher was made by Blaise de Viginere, a French diplomat, to cover up the flaws in the Monoalphabetic Cipher, by using more than one alphabet! This lets it fall into the class of a Polyalphabetic Cipher. Aptly, this cipher is also called the Chiffre Inde(acute)chiffrable (the Indecipherable Cipher!). It was finally broken by Charles Babbage, an English Mathematitian, in 1854.

How it works

It already looks a lot more difficult! The way to use it is to pick a codeword, for example, let us pick the word BLUE. Lets say we wanted to send the message: Good morning. To use the table, we would go down the sides of the columns to the first letter of the codeword. In this case it is B, so we would go to the row immediately below us. Along the top row we would then go along to the letter we wanted to encode, in this case it is the letter g. The intersection of these rows and columns would be the resulting letter we would be enciphered to, so g is changed to h. For the next letter we would take the next letter of the codeword, L, and use that row to encrypt o, and we would carry on like that. So the message would be sent like this:


B L U E B L U E B L U
g o o d m o r n i n g
H Z I H N Z L R J Y A

Do you see how this comes about? Notice that although there is a double o in 'good', each o is enciphered to a different letter! And also notice that the letter H appears twice in the ciphertext, but it represents both a g and a d! This is the brilliance of the Viginere cipher. No wonder it was called the Chiffre Indechiffrable!

How to solve

However, like most ciphers, there is always a flaw ready to be exploited. If the cipher text is long enough, there are a few patterns to observe that make the Viginere cipher vulnerable to attack.

The key to cracking this most fiendish of ciphers is to figure out how long the key is. If this can be done, then we can just come back to our friend 'Frequency analysis', and the rest of the code should be in the bag!

To figure out how long the codeword is, we need to look for patterns. Check out this example, with the codeword LUCKY:

L U C K Y L U C K Y L U C K Y L U C K Y L
t h e s e a t h e s u n a n d t h e s k y
E B G C D L N J O R F H C X B E B G C I J

Did you see how EBGC is repeated twice? This is becuase it overlaps the same part of the codeword! As a general rule, any repetition for more that 4 letters in a row is more likely to be an overlap of a common word than a coincidence! Using this very small piece of information, we can have a guess at how long the codeword is! Counting how far apart these two repetitons are, we can see they are 15 spaces apart. So it would be reasonable to assume that the codeword is 3, 5 or 15 letters long.( Of course it could be 1 letter long, but this is just a Caesar cipher and that is not very secure at all!). If the plaintext was longeer, we could look for more of these repetitions, and look for common multiples between the distances between the two. For instance, if the we had another sequence which was 50 letters apart, then this could have a key length of 2, 5, 10, 25 or 50. This would make 5 a very likely candidate!

Now what? Once we have a suspicion on how long the codeword is, we can split up the ciphertext. The first section would correspond to the first letter of the key, the second, to the second, and so on. What we would do now is perform Frequency analysis on each of these groups of letters separately. Very heplfully, if we are right about the length of the key, the first group simply corresponds to a Caesar Cipher! So if we can build up a picture of the distribution of the letters, we can figure out the Caesar cipher it corresponds to, and figure out the letter of the codeword. If we can do that for each letter of the codeword, or if we can guess the codeword or the message from what we have managed to figure out already, then we have solved the Viginere Cipher!
